# Networker

Networker is the canonical person-intelligence and connection-planning skill.
It combines the best of the upstream Atlas and Networker skills:

- Atlas depth: footprint scoring, mandatory fetches, bridge taxonomy, temporal overlap, Sales Navigator feedback, and connection-plan discipline.
- Networker execution: parallel specialist agents, Wave 2 follow-ups, structured dossier, outreach drafts, and failure handling.
- Web-recon architecture: thin front door, companion docs, runnable CLI, artifact contracts, gates, and tests.

## Safety Boundary

Use public professional sources only. Do not research minors, private individuals
without a public professional footprint, private addresses, personal family data,
real-time location, authenticated content, paywall bypasses, or coercive leverage.
Unknown plus queries tried is better than invention.

Read [ETHICS.md](ETHICS.md) before any high-sensitivity target.

## Canonical Workflow

1. Parse target and create an engagement:
   ```bash
   python3 ~/.claude/skills/networker/networker-cli.py init '"Person Name" --company "Company" --context intro --depth deep --outreach'
   ```
2. Review the config and artifact root under `~/person-intel/{slug}/`.
3. Run Phase 0 and ground-truth loading from [PHASES.md](PHASES.md).
4. Dispatch specialist prompts generated by:
   ```bash
   python3 ~/.claude/skills/networker/networker-cli.py prompts {slug}
   ```
5. Save every agent result as JSON under `~/person-intel/{slug}/agents/`.
6. Maintain `sources.jsonl`, `claims.json`, and `recon-state.json` while researching.
7. Validate before reporting:
   ```bash
   python3 ~/.claude/skills/networker/networker-cli.py validate {slug}
   ```
8. Generate or update the dossier:
   ```bash
   python3 ~/.claude/skills/networker/networker-cli.py report {slug}
   ```

## Modes

| Mode | When | Behavior |
|---|---|---|
| `fast` | Meeting is soon, target is visible | Core agents, minimum viable fetch log, no broad creative sweep unless signals demand it |
| `standard` | Default professional prep | Phase 0 score, ground-truth diff, all core agents, Wave 2 if useful |
| `deep` | High-value target or weak public footprint | Atlas-style exhaustive discovery, full bridge taxonomy, regional/native-script passes, event map |
| `feedback` | Sales Nav or CRM mutuals arrive later | No new research; merge confirmed mutuals and re-rank warm paths |

## Non-Negotiables

- Every factual claim needs a fetched source URL.
- Every source used in the final dossier must appear in `sources.jsonl`.
- Every synthesized claim must appear in `claims.json` with `HIGH`, `MEDIUM`, or `LOW` confidence.
- Every claim source must match a fetched source URL in `sources.jsonl`.
- `agents/connection-plan.json` must include a complete warm `primary_path` before `report` can generate deliverables.
- Network bridges require temporal overlap analysis, not just shared institution names.
- Verified-client and suppression rules apply before any outreach draft.
- Store runtime artifacts under `~/person-intel/{slug}/`, never inside the skill bundle.
